# Contributing to ROS2 DevKit

Thanks for your interest in contributing.

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Push and open a pull request

## Pull Request Guidelines

- Keep PRs focused on a single change
- Follow existing code style
- Update relevant skill documentation if you add or modify skills
- Test your changes with Claude Code before submitting

## Skill Conventions

- Skills use the structure defined in `CLAUDE.md`
- Agent definitions live in `claude-code/agents/`
- Skill definitions live in `claude-code/skills/`
- Scripts live in `claude-code/scripts/` (generators for Behavior-Tree, CMake, context files, and Docker configurations)

## Reporting Issues

Use GitHub Issues for bugs, feature requests, or questions.
