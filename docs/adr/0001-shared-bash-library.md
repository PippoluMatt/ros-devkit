# Extract shared bash library and rename scripts/ to install/

Both `install.sh` and `update.sh` duplicated ~140 lines of identical or near-identical functions (log, die, cleanup, need_command, expand_path, source acquisition, namespace cleaning, CLI wrapper generation). The CLI wrapper had already drifted — `install.sh` emitted `ROS_DEVKIT_LOCAL_SANDBOX` env vars that `update.sh` did not. A bug fix in source acquisition required editing two files. We extracted a shared library (`install/lib/install_common.sh`) sourced by both scripts, and renamed the top-level `scripts/` directory to `install/` to co-locate the library with its consumers. `update.sh` dropped the `ROS_DEVKIT_SOURCE` env override for `SOURCE_DIR`, computing it from `INSTALL_HOME` like `install.sh` does. `install.sh` handles curl-pipe mode by downloading the library to a temp file when no local copy is found. The `install/` directory name collides with colcon's default build output; this is acceptable because the repo is not a ROS2 package workspace.
